# Security Policy

## Supported Versions

Security fixes are prioritized for:

| Version | Supported |
| --- | --- |
| `main` | Yes |
| latest tagged release / App Store build | Yes |
| older snapshots | Best effort |

## Reporting a Vulnerability

Please do not open a public issue for security problems.

Preferred path:

1. Use GitHub's private vulnerability reporting for this repository.
2. If private reporting is unavailable, contact the maintainer privately through GitHub and include `SECURITY` in the subject.

Please include:

- affected commit, tag, or build
- impact and attack preconditions
- clear reproduction steps or proof of concept
- any suggested mitigation if you already have one

## Response Expectations

- Initial acknowledgement target: within 5 business days
- Status update target: within 10 business days

We will coordinate disclosure timing with the reporter when a fix is needed.
